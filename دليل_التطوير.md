# ⚔️ سيف - دليل التطوير

## كيف تعمل سيف من الداخل؟

### 1. المعمارية

```
كود سيف (.سيف)
     │
     ▼
حداد2 (سكريبت Bash)
     │
     ├── 1. يقرأ ملف .سيف
     ├── 2. يشيل أسطر #ادرج
     ├── 3. يحول التوجيهات العربية لإنجليزية
     │      (#عرف ← #define, #ادرج ← #include ...)
     ├── 4. يدمج المكتبات (.ع) مع الكود
     ├── 5. يمرر الملف الناتج لـ GCC
     └── 6. GCC ينتج ملف تنفيذي
```

### 2. نظام الكلمات المفتاحية

كل الكلمات العربية مجرد `#define` في ملفات `.ع`:

```c
// كرتونة.ع
#define عدد      int
#define اذا      if
#define طالما    while
#define هيكل     struct
#define تعريف_نوع typedef
```

### 3. تدفق الترجمة في حداد2

```bash
# 1. إزالة #ادرج وتحويل التوجيهات
grep -v '#ادرج' "$file" | sed 's/#عرف/#define/g' ... > ملف_مجهز.c

# 2. تجميع المكتبات
cat كرتونة.ع > المخرج.c
cat بصمة.ع >> المخرج.c
cat صوت.ع >> المخرج.c
...

# 3. إضافة كود المستخدم
cat ملف_مجهز.c >> المخرج.c

# 4. ترجمة بـ GCC
gcc المخرج.c -o البرنامج -lSDL2 -lSDL2_mixer ...
```

جميع ملفات `.ع` تبدأ بـ `#ifndef` للحماية من التكرار.

## كيفية إضافة كلمة مفتاحية جديدة

**مثال: إضافة كلمة `مصفوفة` كمرادف لـ `array`**

بما أن C ليس لديها type `array` حقيقي، سنضيفها كتعريف نوع:

في `كرتونة.ع`:
```c
// يمكن استخدامها مع مؤشر
#define مصفوفة *
```

أو إذا كنت تريد إضافة type حقيقي:
```c
// لا يمكن عمل #define لأن array في C معقدة
// الأفضل استخدام المؤشرات كما في C العادية
```

## كيفية إضافة دالة مكتبية جديدة

**مثال: إضافة دالة `عكس_نص` (strrev)**

في `نصوص.ع`:
```c
// استخدام دالة C موجودة
#define عكس_نص strrev

// أو كتابة دالة بنفسك
عدم عكس_نص(محرف *نص) {
    عدد ط = طول_نص(نص);
    لكل (عدد ع = 0; ع < ط/2; ع++) {
        محرف مؤقت = نص[ع];
        نص[ع] = نص[ط-1-ع];
        نص[ط-1-ع] = مؤقت;
    }
}
```

## كيفية إضافة مكتبة جديدة

1. أنشئ ملف `.ع` جديد في مجلد `حداد/`

```c
/* ══════════════════════════════════════
   ⚔️  سيف - اسم المكتبة
   ══════════════════════════════════════ */

#ifndef اسم_مكتبة
#define اسم_مكتبة

#include <stdio.h>

// تعريفاتك هنا

#endif
```

2. أضف المكتبة إلى `حداد2`:
```bash
# أضف هذا السطر بعد المكتبات الأخرى
cat "$H/اسم_ملفك.ع" >> "$tmp"
```

## كيفية إضافة توجيه معالج مسبق جديد

في `حداد2`، أضف قاعدة `sed` جديدة:

```bash
sed \
    -e 's/#عرف/#define/g' \
    -e 's/#ادرج/#include/g' \
    -e 's/#توجيهي_الجديد/#pragma/g' \  # ← أضف هنا
    "$file" > ملف_مجهز.c
```

## كيفية كتابة مكتبة رسوميات متقدمة

المكتبات الحالية (`بصمة.ع`, `صوت.ع`) تستخدم `#define` للدوال الموجودة.
لإضافة دوال معقدة (تحتاج لحالة أو منطق):

```c
// في ملف .ع الجديد
#include <SDL2/SDL.h>

// دوال بسيطة بـ #define
#define ابدأ_الرسوم() SDL_Init(SDL_INIT_VIDEO)

// دوال معقدة كدالة C عادية
عدد حسّن_الأداء(راسم *ر) {
    SDL_SetHint(SDL_HINT_RENDER_SCALE_QUALITY, "1");
    return SDL_RenderSetLogicalSize(ر, 800, 600);
}
```

## نصائح للمطورين

1. **اختبر كل تغيير**: `./حداد2 مشاريعي/اختبار.سيف`
2. **لا تكرر التعريفات**: استخدم `#ifndef` في كل مكتبة
3. **أسماء واضحة**: الكلمة العربية تعكس المعنى بدقة
4. **التعليقات**: بالعربية أو الإنجليزية - المهم تكون مفيدة
5. **Git**: commit بعد كل تغيير ناجح
6. **GCC**: أي كود C عادي يعمل في سيف - استخدم مكتبات C مباشرة

## الأخطاء الشائعة وحلولها

| المشكلة | الحل |
|---------|------|
| `بصمة.ع: No such file or directory` | شغّل `حداد2` من داخل مجلد `حداد/` |
| `undefined reference to 'main'` | تأكد من وجود `عدد رئيسي()` |
| `expected ';'` | نسيت `;` - الأخطاء تُترجم للعربية! |
| ملف SDL غير موجود | `sudo apt install libsdl2-dev` |
| الخط لا يظهر | `sudo apt install fonts-scheherazade` |
